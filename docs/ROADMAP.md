# TransitHQ Roadmap

## Phase 1 (MVP)
- Backend proxy to MTS Where API + GTFS-Realtime
- Endpoints: health, nearby stops, stop arrivals, vehicles, alerts
- Frontend: mobile-first PWA-style UI

## Phase 2 (Own data layer)
- GTFS static ingestion to Postgres
- GTFS-Realtime ingestion to Redis + Postgres history

## Phase 3 (Intelligence)
- Arrival confidence scoring
- Headway and bunching detection
- Reliability analytics dashboards
